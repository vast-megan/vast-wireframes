Vast 2026 — Antinomy Design Brief

What we're building

A design system made of reusable building blocks. Every component you create should be able to live in multiple contexts across the site.

Think Apple Design System, not custom illustrations.
Think Stripe component library, not landing pages.

The three pillars

  1. Restraint — Less is more. Strong typography does the work. When in doubt, remove.
  2. Flexibility — Every component must be reusable across at least 3 different page contexts.
  3. Clarity — A visitor should immediately understand what they're looking at. We want to create consistency where we can and let them dig deeper if we allow them to.

Rules that apply to every component

  • No text placed over static images
  • Sharp corners: 0px radius on images
  • Glass/blur effects only over video — never over static images
  • Default alignment is left — center only for XL type moments
  • Limit the type ramp wherever possible
  • + crosshairs as framing devices — intentional, not decorative

Questions to ask for every component you design

  • Can this be built from smaller atoms and molecules we already have?
  • Can this be reused elsewhere?
  • Is this a one-off? If it is, why — and is there a simpler version available?
  • Does this work in both a light and dark context?
  • What's the simplest version of this?
  • How does this behave on mobile?
  • Could this serve 3 or more different page types?
  • If we removed half the content, would it still work?
  • If a user wants to find out more, how can they do that?

What not to focus on

  • Designing full pages — components only
  • If a component is a one-off, be intentional about why and when it would be used

Where to start

Work through components in this order:

  1. Navigation
  2. Footer
  3. Hero
  4. Content sections

Max 2 design rounds per component — then we decide or cut.
